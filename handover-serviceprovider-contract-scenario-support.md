# Handover — RFC 0010 / DF-27: service-provider contracts scenario-testable (uniform)

**Goal.** Make every `[ServiceProviderContractType]` *value* contract — the SDK's own digital/analog I/O
and third-party ones like Ecocoach PPC — drivable/assertable from a committed `*.scenario.json` through
**one** generic DevHost handler and **one** generic step pair, with **no hardcoded contract support in the
core** and **no production-path change**. Delete the four hardcoded HAL step kinds → scenario format **v2**.

## Converged design decisions (read before implementing)

1. **Convention-scan harvesting (the key decision).** The *runtime is already generic*; the DevHost is the only hardcoded place. Mirror the runtime:
   - runtime: `Dale/Program.cs:201` `CreateMqttHandlerActors` → `assemblies.GetConcreteTypes(typeof(IMqttHandlerActor))`, create each under `type.Name`; `Dale/Configuration/LogicSystem/LogicSystemConfigurationInitializer.cs:566` → `GetConcreteTypes(typeof(IServiceProviderHandlerActor))`, fan the link map to each by `type.Name`. (Private repo `C:\_gh\dale`.)
   - DevHost today hardcodes four: create at `DevLogicSystemInitializer.cs:150-153` (`CreateMockHalHandlers`), fan-out at `:294-297` (`LinkContractsWithMockHandlers`).
   - **Do**: generalize both to scan `IServiceProviderHandlerActor`, create one generic stand-in per handler type under `type.Name`, read its `[ScenarioWire]` via `ScenarioWireCodec.ForHandler(handlerType)`, fan the link map to each. Instance-free; no contract-instance harvesting needed.
2. **Open question RESOLVED (from source, no test needed).** `DevConfigurationBuilder.AutoCreateServiceProviders` (`:443`) fabricates a `DevContractMapping` for **every** `[ServiceProviderContractType]` contract, unconditionally; `LinkContractsWithMockHandlers` (`:271-289`) builds the link map from **all** mappings, unfiltered. So a custom contract is already in the map — the generic stand-in "just works" once created + fanned. Still add a **positive round-trip** smoke (not "no throw") — `SendToContractHandler` silently drops on an empty mapping (`LogicBlockContractBase.cs:111-116`).
3. **No production-path change.** The stand-in registers under the contract's *existing* `ContractHandlerActorName` (= `type.Name`), so `LogicBlockBase.cs:115` (`LookupByName`) is untouched. The earlier "retarget override map" idea is dead — don't reintroduce it.
4. **`[ScenarioWire]` on HANDLERS, codec in DevHost.** SDK surface = the declarative attribute only. JSON↔struct lives in the DevHost (`ScenarioWireCodec`), orthogonal to the runtime's FlatBuffer/MQTT path (different side of the handler).
5. **Scope boundary — three IO mechanisms, agreed (put this in the RFC):**
   | mechanism | wiring | examples | verdict |
   |---|---|---|---|
   | value contract | `[ServiceProviderContractType]`, fire-and-forget | di/do/ai/ao, PPC demand | **RFC 0010 MVP** |
   | request/response contract | `[ServiceProviderContractType]` with `Action` callbacks | **Modbus RTU** (`IModbusRtu`) | **deferred** — needs a response-fixture vocab on the same actor plane; NOT `serviceProviderSet` |
   | direct-DI client | injected service, off the actor plane | **HTTP** (`ILogicBlockHttpClient`), **Modbus-TCP** (`ILogicBlockModbusTcpClientFactory`) | **out of scope** — keep Ref*/TestKit; scenarios stay logic+contract-boundary, not protocol mocking |
   The RFC's "Modbus for free" is **wrong** (RTU is request/response).
6. **Format v2.** Delete `digitalInput`/`analogInput`/`digitalOutput`/`analogOutput`; replace with generic `serviceProviderSet { logicBlock, contract }` (drive) / `serviceProviderExpect { logicBlock, contract }` (assert). `set`/`expect`/`waitUntil`/`advance`/`settle`/`wait` stay. Direction from `[ServiceProviderContractType].Consumers` multiplicity (`ZeroOrOne`=output, `ZeroOrMore`=input).

## Gotchas

- **PublicApi snapshot** will drift on `[ScenarioWire]` — regen before the foundation/feature PRs (CI bot auto-commits per [cli-help-snapshot-bot]; don't force-push over it).
- **Namespace shadowing**: DevHost.Test files use the flat `Vion.Dale.DevHost.Test` namespace — a `.Scenarios` sub-namespace shadows the production `Vion.Dale.DevHost.Scenarios` and breaks other files.
- Wire structs must be plain JSON-serializable CLR record structs (value contracts are; Modbus RTU's carry `Action`/`Exception` → request/response, not driveable).
- In the **Bash tool**, `@'...'@` is PowerShell — use a bash heredoc for multi-line commit messages.
